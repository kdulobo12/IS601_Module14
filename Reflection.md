For this assignment, I developed a full-stack FastAPI application that implements BREAD functionality for user-specific calculations. The goal was to allow users to create, view, update, and delete their own calculations while ensuring proper authentication, validation, and database integration.

One of the main challenges I faced was correctly connecting the calculation data to the logged-in user. I had to make sure that each calculation stored a user_id and that all endpoints filtered data based on the current user. This helped ensure that users could not access or modify other users’ data. Another challenge was handling validation and edge cases, such as invalid operations and division by zero. I implemented both backend validation using Pydantic and frontend validation to prevent incorrect inputs. Setting up the testing environment was also challenging. I used pytest for backend testing and Playwright for end-to-end testing. Writing tests for both positive and negative scenarios helped ensure that the application behaved correctly under different conditions. The most difficult part was configuring GitHub Actions and Docker. Initially, I faced issues with Docker image naming and authentication when pushing to Docker Hub. I resolved this by correctly tagging the image and adding Docker Hub credentials as GitHub secrets. I also fixed issues with the Trivy security scan in the CI/CD pipeline.

Through this assignment, I gained a better understanding of how backend APIs, databases, authentication, testing, and deployment work together in a real-world application. I also learned how CI/CD pipelines automate testing and deployment, which is important in modern software development. If I were to extend this project, I would improve the user interface and add more advanced calculations or history tracking features. Overall, this assignment helped me strengthen my understanding of full-stack development, testing, and DevOps practices.
